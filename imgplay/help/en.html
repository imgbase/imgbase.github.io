1. The low resolution of GIF, Why not high quality?

GIF file format has limitation of 256 colors, so there are unavoidable damage to the photo when it is saved in GIF. We recommend saving an image in video format if you want to save a high quality.

The ImgPlay app support to make a gif for the best quality at mobile as it can. The result of gif quality is affected from original's, so if you want to make more good, use the better quality original.

2. Photos loading takes too long

If the photos saved in the device is uploaded to iCloud, loading may take long. The download is through the network, so the process is vulnerable to the network speed.

Try again in Wifi network.

4. Why not sound, Add sound?

The ImgPlay app is focusing on creating the GIF file.
Unfortunately, the GIF file has not any format for the sound in it. It is a unique feature of GIF, so the ImgPlay app does not have a feature of add the sound. To keep the original sound, you must save it to video file format without edit of speed or direction at Edit view.

5. Limitation for frames, Why can not make a long GIF?

Maximum frame limits are depending on the device.

Max 150 Frames : 5s, 6, 6 Plus, SE, 7, 8, iPad
Max 200 Frames : 7 Plus, 8 Plus, X, iPad Pro

The mobile has a small memory than the computer, so when make a GIF using more than 150 frames, the app has been crashing many times due to lack of memory. Therefore there is a limit.
